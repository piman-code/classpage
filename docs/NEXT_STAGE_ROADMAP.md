# classpage 다음 단계 로드맵

이 문서는 현재 구현 범위와 다음 단계 방향을 구분하기 위한 문서입니다.  
핵심은 `지금 이미 있는 것`, `다음 단계`, `나중 단계`를 섞지 않는 것입니다.

현재 기준으로는 `닫힌 베타 / 소규모 실사용 테스트`를 염두에 둔 읽기 중심 범위를 정리합니다.

즉, 이번 릴리즈의 목적은 선생님이 학생 결과를 읽고 다음 행동을 고르게 돕는 화면을 먼저 안정적으로 검증하는 것입니다.

## 1. 제품 방향 원칙

- 먼저 해야 할 행동과 확인 위치를 문서 앞에 둡니다.
- 초보자/고급자 모드를 지금은 나누지 않습니다.
- 실제 선생님이 바로 쓸 수 있는 가벼운 흐름을 우선합니다.
- 큰 아키텍처 변경보다 문서 정리와 얇은 UX 분리를 먼저 합니다.
- AI 인사이트와 자동 경보는 필수가 아니라 선택 기능으로 둡니다.

## 2. 지금 이미 있는 것

지금 바로 쓰는 범위는 아래입니다.

- 학생용 페이지
  오늘의 할 일, 공지사항, 학급용/수업용 Google Form 버튼을 설정값으로 표시합니다.
- 선생님 페이지
  집계 연결 상태와 함께 학급 관리, 수업 관리, 별점 영역을 빠르게 확인합니다.
- 선생님 화면 분리 방식
  한 화면 안에서 보더라도 학급 관리, 수업 관리, 별점 영역을 분리해서 보게 합니다.
- 핵심 운영 UX 정리
  대상 학급 배지, 과목+단원+최근 수업 범위+수업 그룹 탐색, 수업 우선 확인 카드, 다음 피드백 대상 정리, 별점 규칙별 발생 현황과 학생 흐름 필터를 읽기 전용으로 확인합니다.
- 베타 안내 정리
  빈 상태, 집계 누락, 최근 이벤트 부족, 학생 검색 결과 없음 같은 상황에서 왜 비어 보이는지와 다음 확인 위치를 더 자연스럽게 안내합니다.
- 학생 공개용 템플릿
  [docs/templates/STUDENT_PUBLIC_SHARE_NOTE_TEMPLATE.md](./templates/STUDENT_PUBLIC_SHARE_NOTE_TEMPLATE.md)로 오늘 공지와 공개 가능한 별점 정보를 수동 공유할 수 있습니다.
- 별점/공개 경계
  공개 가능한 기준과 안내문 템플릿은 이미 있지만, 학생 공개용 결과를 자동으로 내보내는 기능은 아직 없습니다.
- 집계 방식
  Google Form -> Google Sheets -> Google 로그인 이메일 기준 집계 -> Apps Script -> JSON -> classpage 흐름을 유지합니다. allowlist는 필요할 때만 보조 기능으로 둡니다.
- 읽기 전용 계약 보강
  `lesson-summary.json`의 `subjectSummaries[].groups[]`, `lessonDate/periodOrder/unitKey/lessonKey`, `star-ledger.json`의 `ruleSummary`를 읽어 다과목/다그룹 수업 탐색, 최근 수업 프리셋, 규칙별 별점 운영 확인과 학생별 최근 흐름 확인을 시작했습니다.

## 3. 다음 단계

다음 단계에서 가장 먼저 추진할 항목은 아래입니다.

- 학생 공개 출력 강화
  오늘의 공지, 오늘 할 일, 공개 가능한 별점 정보를 학생용 공유 노트나 ShareNote 친화 형식으로 더 쉽게 내보내는 흐름을 붙입니다. 지금은 수동 템플릿이 있고, 다음 단계는 자동 생성 또는 반자동 복사 흐름입니다.
- 선생님 화면 분리 강화
  학급 관리 페이지와 수업 관리 페이지를 분리하는 방향으로 갑니다. 현재는 `전체 보기 / 학급 관리 / 수업 관리 / 별점` 토글로 한 화면 안에서 나누고 있고, 다음 단계에서 실제 페이지 분리를 검토합니다.
- 가벼운 운영 체크
  선생님이 지금 무엇을 먼저 봐야 하는지 바로 알 수 있도록 연결 상태, 최근 집계 시각, 섹션별 확인 동선을 더 분명히 합니다.
- 수업/별점 drill-down 확장
  현재는 과목 선택 + 단원/최근 수업 범위/날짜 필터 + 수업 그룹 선택 탐색, 과목별 최근 N개 group 제한, 가벼운 학생 필터와 최근 이벤트 미리보기까지 지원합니다. 다음 단계에서는 단원 비교, 학생별 전체 이력 drill-down, 날짜 범위 확장을 검토합니다.

## 4. 나중 단계

지금 당장 만들지는 않지만 방향을 잊지 않아야 하는 항목입니다.

- 학급 관리 개선
  타임라인/추세 보기, 학생별 드롭다운, 반 전체 감정 추세, 개별 학생 감정 변화
- 수업 관리 개선
  단원 기반 이해도 차트, 학생별 단원 진행도, 단원별 정답/오답 수
- 학생 공개 출력 자동화
  수동 템플릿을 넘어서 classpage 설정값과 공개 집계 결과를 묶어 내보내는 얇은 출력 경로

## 5. 선택 기능

있으면 좋지만 현재 MVP와 다음 단계의 필수 조건은 아닙니다.

- AI 인사이트
  학급 분위기 요약, 수업 이해도 해석, 추천 후속 조치 같은 자동 해석 기능
- 위험 학생 알림
  Google 또는 외부 알림 채널로 보내는 방식 우선 검토
- Obsidian 내부 완결형 알림
  지금은 우선순위가 낮고, 외부 알림과 비교해 본 뒤 결정

## 6. 지금 하지 않는 것

이번 단계에서 의도적으로 하지 않는 것은 아래입니다.

- 초보자/고급자 모드 분리
- 대규모 차트 시스템
- AI 기반 해석을 전제로 한 화면 설계
- 복잡한 알림 아키텍처
- classpage 안에서 Google Sheets 직접 집계

## 7. 정리해서 보면

- 지금 이미 있는 것
  action-first 문서 흐름, 수동 공개 템플릿, 선생님 화면 토글 기반 분리, 대상 학급/다과목 수업 탐색/별점 운영 확인 UX
- 다음 단계
  자동 공개 출력, 더 분명한 선생님 화면 동선, 단원 비교 및 전체 이벤트 drill-down 검토
- 나중 단계
  추세 보기, 고급 차트, AI 해석, 복잡한 알림
